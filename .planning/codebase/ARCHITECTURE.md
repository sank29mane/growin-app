# Architecture Analysis

> Generated by /gsd-map-codebase on 2024-05-22
> Focus: Data Workflow, API Providers (T212, Alpaca, Finnhub), and Agent Functionality.

## Overview

Growin is a sophisticated financial intelligence platform built with a **FastAPI** backend and a **SwiftUI** frontend. It leverages a **Multi-Agent System (MAS)** for autonomous financial analysis and decision-making, optimized for **Apple Silicon (M1-M4)** hardware.

## 1. Data Workflow & API Providers

The system employs a multi-tiered data acquisition strategy with region-based routing and resilient fallbacks.

### API Providers
| Provider | Role | Key Components |
|----------|------|----------------|
| **Alpaca Markets** | US Primary Source | `AlpacaClient` in `data_engine.py`, `market_routes.py` |
| **Finnhub** | UK Primary Source | `FinnhubClient` in `data_engine.py`, `utils/data_frayer.py` |
| **Trading 212** | Portfolio & Execution | `trading212_mcp_server.py`, `t212_handlers.py` |
| **Yahoo Finance** | Universal Fallback | `AlpacaClient._fetch_from_yfinance` |

### Data Acquisition Flow
1. **Request**: A request for market data (OHLCV, quotes) is initiated by an agent or a route.
2. **Normalization**: The ticker is normalized using `utils/ticker_utils.py` (e.g., `VOD` -> `VOD.L` for UK).
3. **Routing**:
   - **US Stocks**: Routed to **Alpaca**.
   - **UK Stocks**: Routed to **Finnhub** (using `.L` suffix detection).
4. **Resilience**:
   - **Circuit Breakers**: `resilience.py` monitors API health and trips if failure thresholds are met.
   - **Fallbacks**: If the primary source fails, the system automatically falls back to **yfinance**.
5. **Currency Alignment**: UK stock prices (often in GBX/pence) are normalized to GBP in `utils/currency_utils.py`.
6. **Caching**: Data is cached in `cache_manager.py` with configurable TTLs.

## 2. Agent Functionality

The platform uses a **Multi-Agent System (MAS)** where specialized agents collaborate to provide insights.

### Agent Core Architecture
- **`BaseAgent`**: Abstract base class defining common communication and lifecycle methods.
- **`OrchestratorAgent`**: The "Central Brain" that classifies user intent and delegates tasks to specialized agents.
- **`Messenger`**: Handles inter-agent communication and logging of "thought traces."

### Specialized Agents
- **`QuantAgent`**: Analyzes technical indicators (ORB, RSI, MACD) and statistical patterns.
- **`ResearchAgent`**: Performs RAG-enhanced market research using web search and news APIs.
- **`RiskAgent`**: Evaluates portfolio exposure and market volatility regimes.
- **`ForecastingAgent`**: Uses ML models (including local MLX/CoreML) for price prediction.
- **`WhaleAgent`**: Monitors large institutional trades and "smart money" movements.
- **`DecisionAgent`**: Synthesizes all agent signals into a final verdict or autonomous action.

### Agent Workflow
1. **Intent**: `Orchestrator` receives a query (e.g., "Analyze VOD.L").
2. **Delegation**: `Orchestrator` spawns/tasks `QuantAgent` (data), `ResearchAgent` (news), and `RiskAgent`.
3. **Execution**: Agents perform their specialized tasks, interacting with the `DataEngine` or external tools.
4. **Synthesis**: Results are passed back to the `DecisionAgent` for final evaluation.
5. **Output**: The final insight is streamed to the UI via SSE (Server-Sent Events).

## 3. Technology & Hardware Integration

- **Inference Engines**:
  - **MLX**: Used for local LLM and model weights adaptation on Apple Silicon GPUs.
  - **CoreML / ANE**: Used for high-performance Neural JMCE inference on the Apple Neural Engine.
- **Performance**: Rust-based core (`growin_core_src`) handles compute-intensive data processing.
- **Integration**: MCP (Model Context Protocol) provides a standardized way for agents to interact with external tools (T212, Docker, HF).
