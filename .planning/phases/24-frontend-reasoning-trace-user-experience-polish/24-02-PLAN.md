# Plan 24-02: Metal-Accelerated UI Polish & 120Hz Rendering Optimization

This plan focuses on implementing the final visual polish for the reasoning trace and optimizing the rendering pipeline for 120Hz displays on Apple Silicon.

## 1. Metal-Accelerated NPU Glow
Currently, the NPU glow effect on the `ReasoningStepChip` uses native SwiftUI `.blur` and `.blendMode`. We need to determine if a custom Metal shader (`.metal`) is necessary for performance or aesthetic reasons, or if the native SwiftUI approach is sufficient for the 120Hz budget.

*   **Task**: Profile the current SwiftUI `.blur` + `.blendMode` implementation using Instruments (Core Animation FPS).
*   **Decision**: If FPS drops below 120Hz during the `.symbolEffect` pulsing, replace the SwiftUI blur with a custom Metal Shader (`NPUGlow.metal`).
*   **Implementation (If Metal is chosen)**:
    *   Create `Growin/Shaders/NPUGlow.metal`.
    *   Implement a `[[ stitchable ]]` fragment shader that generates a pulsating cyan/blue aura based on a time uniform.
    *   Update `ReasoningStepChip` to use `.colorEffect(ShaderLibrary.npuGlow(...))`.

## 2. 120Hz Smoothness & Edge Case Optimization
The `ChatView` and `LMStudioViewModel` have been optimized, but we need to audit remaining edge cases that could cause micro-stutters during high-speed scrolling or rapid state updates.

*   **Task**: Audit `ChatView` scroll performance when `activeReasoningSteps` updates rapidly (e.g., 10+ steps per second).
*   **Implementation**:

    *   Check for any forced layout recalculations in `ChatReasoningTraceView` during state transitions.
    *   Review `InteractiveChartView` (if visible during streaming) to ensure its updates don't block the main thread.
