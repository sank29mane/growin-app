# 25-01-PLAN: Institutional Risk Infrastructure & Data Integration

## Objective
Implement the data and logic layers required for institutional risk governance, including liquidity modeling and macro-economic triggers.

## Step 1: Market Context Enhancement
- **File**: `backend/market_context.py`
- **Action**: Add `RiskGovernanceData` Pydantic model and integrate it into `MarketContext`.
```python
class RiskGovernanceData(BaseModel):
    vix_level: Optional[Decimal] = None
    yield_spread_10y2y: Optional[Decimal] = None
    adv_30d: Optional[Decimal] = None
    liquidity_impact: Optional[Decimal] = None  # (order_size / adv_30d)
    systemic_risk_level: str = "NORMAL"  # NORMAL, ELEVATED, EXTREME
```

## Step 2: ADV & Liquidity Logic in QuantEngine
- **File**: `backend/quant_engine.py`
- **Action**: Implement `calculate_adv_30d` and `estimate_liquidity_impact`.
- **Logic**: Use `Decimal` for precision. ADV = sum(volumes) / 30.

## Step 3: Macro-Economic Data Fabrication
- **File**: `backend/data_fabricator.py`
- **Action**: Implement `_fetch_macro_indicators` fetching `^VIX`, `^TNX` (10Y), and `^IRX` (13W/3M).
- **Fallback**: Neutral values if API fails.

## Step 4: Risk Agent Integration
- **File**: `backend/agents/risk_agent.py`
- **Action**: 
    - Extract `order_size` from `suggestion` string using Regex.
    - Compare `order_size` to `adv_30d`.
    - Update prompt to include VIX and Spread data.
    - Enforce mandatory `FLAGGED` status for VIX > 30 or Spread < 0.

## Step 5: ACE Score Penalization
- **File**: `backend/agents/ace_evaluator.py`
- **Action**: 
    - Apply `0.8` multiplier if `vix_level` > 30.
    - Apply `0.9` multiplier if `yield_spread_10y2y` < 0.
