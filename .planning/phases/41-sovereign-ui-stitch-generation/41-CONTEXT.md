# Phase 41: Sovereign UI - Stitch Generation & UX Refinement - Context

## Phase Objective
Create a high-density, professional trading environment using **Stitch MCP tools** that implements the **Sovereign Ledger** (Brutal Editorial) aesthetic. This phase transitions from core primitives to full-screen functional layouts inspired by Trading 212's density and the Growin App's authoritative character.

## Core Aesthetic: Sovereign Ledger (Brutal Editorial)
- **Corner Radius:** Strict **0px** (sharp edges only).
- **Colors:**
  - Background: `#131313` (Main), `#1C1B1B` (Tonal Layering).
  - Accents: `#DFFF00` (Electric Chartreuse) for Alpha/Positive Change.
  - Borders: `1px` subtle tonal shifts or dotted underlines.
- **Typography:**
  - **Noto Serif:** For editorial titles and authoritative statements.
  - **Space Grotesk:** For technical data and tabular numbers.
  - **Monaco:** For agent reasoning traces and system logs.
- **Layout:** "Topological Betrayal" (Radical asymmetry, extreme margins).

## Locked Decisions (from Phase 40 & Research)
- **D-41-01:** Mandatory 120Hz smooth performance on M4 Pro (SwiftUI/Swift Charts).
- **D-41-02:** Use Stitch MCP for all initial screen generation to maintain "Design DNA."
- **D-41-03:** No "Bento" grids; use modular, high-density ledger lists.
- **D-41-04:** Execution must involve a "Slide-to-Confirm" or "Hard-HITL" (Human-In-The-Loop) mechanism.
