# Phase 29: Institutional Portfolio Optimization (Mean-Variance)

## Objective
Implement Modern Portfolio Theory (MPT) via mean-variance optimization and efficient frontier calculations. Integrate these capabilities into the existing `QuantEngine` and `DataFabricator` to provide institutional-grade portfolio rebalancing recommendations.

## 1. Core Mathematical Models (PortfolioAnalyzer)
- **File**: `backend/utils/portfolio_analyzer.py`
- **Focus**:
    - **Covariance & Expected Returns**: Add methods to compute the annualized covariance matrix and expected returns vector from historical daily returns.
    - **Mean-Variance Optimization**: Implement `optimize_portfolio_weights` using `scipy.optimize.minimize`. Support multiple objective functions:
        - Maximum Sharpe Ratio (Tangency Portfolio).
        - Minimum Volatility Portfolio.
        - Target Return with Minimum Variance.
    - **Efficient Frontier Generation**: Create `calculate_efficient_frontier` to generate a curve of optimal risk-return portfolios (array of target returns and their minimized volatilities).
    - **Constraints & Bounds**: Support weight bounds (e.g., $0 \le w_i \le 1$ for long-only) and ensure weights sum to 1.0 ($\sum w_i = 1$).

## 2. QuantEngine Integration
- **File**: `backend/quant_engine.py`
- **Focus**:
    - Add an `optimize_portfolio` method acting as the public interface for the new optimization math.
    - It should accept a list of positions (with historical price series) or rely on `DataFabricator` outputs.
    - Convert numerical outputs to precision-safe `Decimal` where applicable for consistency with financial math standards in the codebase.
    - Handle edge cases: collinear assets, insufficient history, or single-asset portfolios gracefully by returning the current allocation or a neutral response.

## 3. DataFabricator & DataEngine Enhancements
- **File**: `backend/data_fabricator.py` & `backend/data_engine.py`
- **Focus**:
    - For portfolio-wide optimization, historical price data is required for *all* assets simultaneously to compute the covariance matrix accurately.
    - Add a specialized intent (e.g., `portfolio_optimization`) in `DataFabricator.fabricate_context` that orchestrates parallel fetching of historical price series for the entire portfolio universe.
    - Implement caching for the correlation/covariance matrices since they don't change drastically intraday.

## 4. Agent Orchestration
- **File**: `backend/agents/portfolio_agent.py` or a new `backend/agents/optimization_agent.py`
- **Focus**:
    - Integrate the optimization request into the agent workflow.
    - The agent will evaluate the current `PortfolioData`, invoke the `QuantEngine` for the optimal weights, and compare the current allocations against the optimal ones.
    - Generate a rebalancing plan with specific buy/sell actions to transition the user's portfolio towards the optimal mean-variance state, utilizing the existing `analyze_rebalancing_opportunity` method in `QuantEngine`.

## 5. System Robustness & Testing
- **Files**: `tests/backend/test_portfolio_optimization.py`
- **Focus**:
    - Unit tests for covariance calculation and optimization algorithms using deterministic synthetic data or known historical datasets.
    - Ensure `scipy` optimization constraints work as expected (weights sum to 1, bounds respected).
    - Handle fallback mechanisms if `scipy` is unavailable (though Scipy is available in standard environments).
