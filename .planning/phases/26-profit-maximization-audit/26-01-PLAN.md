# 26-01-PLAN: Core Profitability Audit & Agent Calibration

## Objective
Audit and re-calibrate the MAS pipeline to ensure it actively generates actionable, profit-maximizing intraday and intra-week trading plans.

## Step 1: Intraday Data Velocity Audit
- **Target**: `backend/data_fabricator.py`, `backend/agents/orchestrator_agent.py`
- **Action**: 
    - Review how `OrchestratorAgent` passes timeframe resolution to `DataFabricator`. Currently, it defaults heavily to daily data.
    - Implement a mechanism where intents like `intraday_trade` or `swing_trade` automatically fetch `5Min` or `1Hour` data for the `QuantAgent`.

## Step 2: Prompt Engineering for Profit
- **Target**: `backend/agents/decision_agent.py`, `backend/agents/orchestrator_agent.py`
- **Action**: 
    - Update the System Prompts.
    - Shift persona from "Financial Advisor" to "Aggressive, Calculated Swing/Day Trader".
    - Force output to strictly follow a specific format:
        - **Target Entry**: [Price]
        - **Take Profit**: [Price] (Targeting X% gain)
        - **Stop Loss**: [Price] (Max X% loss)
        - **Conviction Level**: [Score]

## Step 3: Risk Governance Calibration
- **Target**: `backend/agents/risk_agent.py`, `backend/agents/ace_evaluator.py`
- **Action**:
    - Audit the `RiskAgent` to ensure the Phase 25 macro triggers (like VIX > 30) do not outright ban *shorting* or *volatility trading* (like options or inverse ETFs), which are highly profitable in such environments.
    - Add a `trade_horizon` variable to `MarketContext` so the Risk Agent knows if the position will be held for 2 hours vs 2 months (adjusting margin/exposure checks accordingly).

## Step 4: Backtesting Pipeline (The "Show Me The Money" Test)
- **Target**: `tests/backend/test_profit_audit.py`
- **Action**:
    - Create a test script that feeds historical high-volatility intraday data (e.g., AAPL on an earnings day) into the `OrchestratorAgent` and measures whether the resulting "Trading Plan" successfully captured the move (simulated PnL).
    - Measure processing latency: Must be < 3 seconds to be useful for intraday.

## Step 5: Verification
- Execute `test_profit_audit.py`.
- Verify the output format contains explicit Entry/Exit/SL coordinates.
- Ensure the MLX Quant Engine handles `5Min` resolution flawlessly.
