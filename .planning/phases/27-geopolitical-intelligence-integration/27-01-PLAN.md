# 27-01-PLAN: LM Studio Reliability & Functional Audit

## Objective
Implement a comprehensive audit and testing suite for `LMStudioClient` to ensure robust local inference on M4 Pro/Max hardware.

## Step 1: Reliability Test Suite Implementation
Create `tests/backend/test_lm_studio_audit.py` with the following test cases:
- **RAM-RULE-01**: Verify `_setup_concurrency_limits` correctly calculates slots based on total RAM.
- **LOAD-01**: Test `ensure_model_loaded` correctly identifies loaded vs. unloaded models.
- **RECOVERY-01**: Simulate a "Channel Error" (HTTP 500 with specific error body) and verify the recovery path triggers `unload_model` -> `load_model` -> `retry`.
- **STATEFUL-01**: Verify `stateful_chat` returns a `response_id` and correctly passes `previous_response_id` in subsequent calls.

## Step 2: Edge Case Prompt Audit (Dry Run)
Add a `dry_run` mode to `LMStudioClient` that skips the network call and returns mock responses for rapid edge-case testing:
- **EMPTY-PROMPT**: Send empty user message.
- **OVERFLOW-TOKENS**: Send messages exceeding context window.
- **MALFORMED-TOOL**: Send a mock tool call that is missing required arguments.

## Step 3: LLMFactory Hardening
Update `LLMFactory._create_lmstudio` to handle the case where `ensure_model_loaded` succeeds but the model is still initializing (polling or wait logic).

## Step 4: Verification
- Execute `uv run python tests/backend/test_lm_studio_audit.py`.
- Verify that the memory guard slots are correctly logged for the current machine's RAM.
- Ensure the recovery path successfully retires the chat request after a simulated crash.
