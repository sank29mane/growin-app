# 27-02-PLAN: Global Monitor Bridge & Geopolitical NewsRAG

## Objective
Integrate Geopolitical Risk (GPR) indices and specialized geopolitical news aggregation into the MAS to provide macro-aware trading signals.

## Step 1: Geopolitical Data Model
- **File**: `backend/market_context.py`
- **Action**: Add `GeopoliticalData` and `GeopoliticalEvent` Pydantic models.
- **Fields**: 
    - `gpr_score`: Normalized score (0.0 to 1.0).
    - `global_sentiment_label`: BULLISH, BEARISH, VOLATILE, CRISIS.
    - `top_events`: List of `GeopoliticalEvent` (title, impact, region).

## Step 2: Geopolitical Agent Implementation
- **File**: `backend/agents/geopolitical_agent.py`
- **Action**: Create a new agent that:
    - Fetches "Geopolitical Risk" and "Global Conflict" news via `NewsDataIOClient` and `TavilyClient`.
    - Uses a small LLM (e.g., `granite-tiny`) to synthesize a `gpr_score` based on headlines.
    - Specifically looks for: Trade Wars, Central Bank shifts, Regional Conflicts, and Sanctions.

## Step 3: Geopolitical RAG Integration
- **File**: `backend/rag_manager.py`
- **Action**: Add `add_geopolitical_event` method to index global events.
- **Logic**: Store with `type="geopolitical_event"` and `impact_score`.
- **Retrieval**: Allow the `DecisionAgent` to query the last 30 days of geopolitical events for context.

## Step 4: DataFabricator & DecisionAgent Update
- **File**: `backend/data_fabricator.py`
- **Action**: Add `_fetch_geopolitical_data` to `fabricate_context`.
- **File**: `backend/agents/decision_agent.py`
- **Action**: Update the system prompt to explicitly weigh `geopolitical_data` when making "Invest" or "Swing" recommendations.

## Step 5: Verification
- Create `tests/backend/test_geopolitical_integration.py`.
- Mock a "High Risk" geopolitical event (e.g., "Suez Canal Blockage") and verify the `DecisionAgent` raises a `FLAGGED` or `BLOCKED` status for affected sectors.
- Verify `gpr_score` is correctly propagated to the SwiftUI frontend (via `MarketContext` dump).
