# 29-01-PLAN: Multi-Asset Bridge (Options & FX)

## Objective
Expand the Growin App MAS beyond equities into Options, Crypto, and Foreign Exchange (FX) to enable more complex hedging and high-yield strategies.

## Step 1: Multi-Asset Data Models
- **File**: `backend/market_context.py`
- **Action**: 
    - Define `AssetType` Enum: `EQUITY`, `CRYPTO`, `OPTION`, `FX`.
    - Enhance `PriceData` or create `MultiAssetPrice` to include:
        - For Options: `strike`, `expiry`, `contract_type` (CALL/PUT), `greeks` (delta, gamma, theta).
        - For FX: `base_currency`, `quote_currency`.
- **File**: `backend/data_models.py`
    - Mirror these changes for persistence and shared typing.

## Step 2: DataEngine Expansion
- **File**: `backend/data_engine.py`
- **Action**: 
    - Implement `get_crypto_bars` and `get_option_bars` using Alpaca's `CryptoHistoricalDataClient` and `OptionHistoricalDataClient`.
    - Implement `get_fx_rates` via Finnhub or yfinance (as fallback).
    - Update `AlpacaClient` to handle these new client types.

## Step 3: DataFabricator Multi-Asset Routing
- **File**: `backend/data_fabricator.py`
- **Action**: 
    - Update `fabricate_context` to detect asset type from ticker format (e.g., `BTC/USD` for crypto, `AAPL240621C00150000` for options).
    - Route to the correct `DataEngine` method based on asset type.

## Step 4: Specialist Agent Updates
- **File**: `backend/agents/research_agent.py`
    - Add crypto-specific and macro-FX news fetchers.
- **File**: `backend/agents/risk_agent.py`
    - Adjust exposure limits for Crypto (higher volatility margin).
    - Add basic options risk check (e.g., "Don't recommend buying deep OTM options with < 7 days to expiry").

## Step 5: Verification
- Create `tests/backend/test_multi_asset_bridge.py`.
- Mock a Crypto price fetch (`BTC/USD`) and verify `DataFabricator` returns a valid `CRYPTO` typed response.
- Mock an Option contract and verify Greeks are correctly populated in the `MarketContext`.
- Ensure `DecisionAgent` prompt reflects the specific asset type (e.g., "Analyze BTC/USD" vs "Analyze AAPL").
